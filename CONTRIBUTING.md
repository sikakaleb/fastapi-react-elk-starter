# Guide de contribution

Merci de votre intérêt pour contribuer à ce projet !

## Comment contribuer

### Signaler un bug

1. Vérifiez que le bug n'a pas déjà été signalé dans les issues
2. Créez une nouvelle issue avec :
   - Un titre clair et descriptif
   - Une description détaillée du problème
   - Les étapes pour reproduire le bug
   - Le comportement attendu vs le comportement actuel
   - Votre environnement (OS, versions des outils, etc.)

### Proposer une amélioration

1. Vérifiez que l'amélioration n'a pas déjà été proposée
2. Créez une nouvelle issue avec :
   - Un titre clair
   - Une description détaillée de l'amélioration
   - Les cas d'usage et bénéfices

### Soumettre une Pull Request

1. Fork le repository
2. Créez une branche pour votre fonctionnalité :
   ```bash
   git checkout -b feature/ma-fonctionnalite
   ```
3. Faites vos modifications
4. Testez vos changements
5. Committez vos modifications :
   ```bash
   git add .
   git commit -m "Description claire de vos changements"
   ```
6. Push vers votre fork :
   ```bash
   git push origin feature/ma-fonctionnalite
   ```
7. Ouvrez une Pull Request sur GitHub

## Standards de code

- Suivez les conventions de nommage du projet
- Ajoutez des commentaires quand nécessaire
- Assurez-vous que le code passe les linters
- Testez vos modifications avant de soumettre

## Questions

Si vous avez des questions, n'hésitez pas à ouvrir une issue pour discuter.
